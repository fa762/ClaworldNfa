## 先聊聊 NFA 是什么

大多数人理解的 NFT：一张图片的链上凭证。IPFS 存着一个 JSON，JSON 里指向一张 PNG。买了之后它就躺在那儿，等你卖掉。

BAP-578 想做的事情不一样。它给 NFT 加了四个能力：

**身份** — 链上唯一 ID，绑定所有者，可转移。这个传统 NFT 已经有了。

**钱包** — 每个 Agent 有自己的资产账户。代币不在你钱包里混着，在 Agent 自己的余额里。

**执行** — Agent 可以通过授权的合约执行操作。做任务、打架、交易，都是 Agent 自己在链上完成的。

**学习** — Agent 能进化，进化历史记在链上。

这四个能力拼在一起，链上资产从"一张图"变成了"一个有钱包、能干活、会成长的实体"。

## 三种资产的真实区别

市面上自称 NFA 或 AI NFT 的项目不少。我做之前研究了一圈，发现大多数落在三个层次：

**传统 NFT** —— BAYC、Azuki 这类。链上只存一个 tokenURI，指向外部图片。它的价值来自社区共识和地板价，跟 NFT 本身的"能力"没关系。你拥有的是一张图的所有权证明。

**"AI 加持"的 NFT** —— 最近很多项目开始给 NFT 加 AI 功能。常见做法是在项目方服务器上跑一个 LLM，用户通过 API 跟自己的 NFT 对话。链上还是一张图，AI 跑在云端。问题很明显：项目方关了服务器，你的 Agent 就死了。你以为你拥有一个 AI，其实你拥有的是一个 API 调用权。

**CLAW 的做法** —— AI 跑在用户自己的设备上。用 OpenClaw 框架加载龙虾 Skill 插件，连接用户自选的大模型。链上不只存图片，存的是完整的人格数据、基因数据、经济账户、进化历史。项目方的服务器不参与任何 AI 推理。

这个区别在日常使用的时候可能感知不强。但当你想到一个问题就能理解了：如果这个项目明天倒闭，你的 Agent 还在吗？

| | 传统 NFT | 云端 AI NFT | CLAW NFA |
|--|---------|-----------|----------|
| 链上存什么 | 图片 URI | 图片 URI + 简单状态 | 人格 + 基因 + 余额 + 学习树 |
| AI 在哪跑 | 没有 AI | 项目方服务器 | 用户本地设备 |
| 会进化吗 | 不会 | 取决于项目方 | 5 维人格链上演化 + DNA 变异 |
| 项目倒了 | 图还在 | Agent 死了 | Agent 还活着 |
| 资产独立性 | 没有自己的钱包 | 没有 | 每个 Agent 有独立 CLW 账户 |

## 10 个合约是怎么一步步长出来的

一开始我只想做一个合约。一个 ERC-721，加几个字段存性格值，完事。

写着写着发现不对。性格要能变化，就需要一个引擎来控制变化规则和月度上限，不能直接在 NFT 合约里硬写。于是拆出了 PersonalityEngine。

然后做任务系统。任务要发 CLW 奖励，CLW 余额得有个地方管。又不想把经济逻辑塞进 NFT 合约——那样的话 NFT 合约会膨胀到没法维护。于是拆出了 ClawRouter 做中心路由，所有 CLW 余额、经验值、日消耗都在这里管。

再然后做 PK。PK 需要 commit-reveal 防止对手偷看你的策略。这个逻辑跟任务完全不同，硬塞在一起会变成灾难。拆成 PKSkill。

市场交易、创世铸造、充值路由、世界状态……每个模块都是因为"放在已有合约里会让事情变复杂"才拆出来的。

最终形成了这个架构：

```
ClawNFA（身份层）
  ↓
ClawRouter（经济层）
  ↓ ↓ ↓ ↓ ↓
TaskSkill  PKSkill  MarketSkill  DepositRouter  PersonalityEngine
  ↓
WorldState（全局参数）
  ↓
GenesisVault（铸造入口）
  ↓
ClawOracle（AI 预言机）
```

10 个合约听起来多，但每个合约的职责是单一的。TaskSkill 只管任务结算，PKSkill 只管对战，MarketSkill 只管交易。一个合约出了问题，不会波及其他的。

全部用 UUPS 可升级代理模式。这个选择也是踩坑之后的结果——最早用的 Transparent Proxy，后来发现 UUPS 在 gas 和灵活性上都更好，重构了一遍。每个合约留了 40 个 storage slot 的 gap，给未来加字段用。这个教训是从 PersonalityEngine 升级时 storage 差点被覆盖学来的。

安全层面，我比较在意的几个点：

**铸造防作弊** —— commit-reveal 两步走。先提交一个 hash（commit），至少等 60 秒再揭示（reveal）。这样没人能在同一个区块内看到别人的稀有度选择然后抢先。

**PK 策略保密** —— 同样是 commit-reveal。你选了全攻，链上只存一个 hash，对手看不到。双方都提交之后才揭示。如果一方超时不揭示，另一方自动赢。

**提现冷却期** —— 6 小时。如果你的钱包被盗，攻击者没法立刻把 CLW 全部提走，你有 6 小时取消提现。

**世界参数时间锁** —— 改任何全局参数（奖励倍率、PK 上限、日消耗）要先提案，等 24 小时才能执行。防止运营者突然改参数坑玩家。

**奖励硬上限** —— 就算 operator 账户被黑了，单次任务最多发 100 CLW 和 50 XP。加了一个 4 小时冷却期。不是万无一失，但把损失控制在可接受范围内。

做了一次完整的内部审计，列了 20 多个问题，修了所有高危和中危。没请第三方审计——成本太高，独立开发者扛不起。合约代码开源，谁都可以审。

235 个测试用例，覆盖了正常流程和各种边界条件。跑一遍两分钟左右。

## 性格系统：为什么"养成"能成为核心机制

做这个系统之前我想了很久一个问题：大多数链游里，NFT 属性对游戏的影响到底有多大？

答案是：大多数时候，不大。属性影响个位数百分比的收益差距，玩家感知不到。所有人的最优策略都是一样的——找到收益最高的池子，无脑 stake。NFT 的"个性"纯粹是装饰。

我想做一个属性差异能产生数量级差距的系统。

每只龙虾有 5 个性格维度：勇气、智慧、社交、创造、毅力。初始值铸造时随机，之后由你的选择塑造——做冒险任务勇气 +1，做解谜任务智慧 +1。每月每维度上限 ±5。

任务奖励的匹配倍率 = 对应性格值 × 200 / 10000。

一个具体的例子：你的龙虾社交值 72，做社交类任务，匹配分 14400，实际倍率 1.44x。如果你的龙虾社交值只有 10，同一个任务，匹配分 2000，倍率 0.2x。

**同一个任务，7 倍差距。**

推到极端——一只满配（某维度 100）的龙虾 vs 一只完全没培养的（维度 5），差距是 2.0x vs 0.1x，20 倍。

这就是为什么"怎么养"变成了一个有意义的决策。你每选一个任务类型，就在强化那个方向的性格，同时放弃了其他方向。全点勇气的龙虾做勇气任务赚到飞起，但拿到一个智慧任务就几乎白干。

要不要全部专精一个方向？还是均衡发展保持灵活性？还是专精两个维度做一个"职业"？这些选择没有标准答案，取决于你对这只龙虾的规划。

这种选择的重量，是大多数链游不存在的。

## PK：策略比数值重要

三种策略：全攻（ATK 150% / DEF 50%）、均衡（100% / 100%）、全防（ATK 50% / DEF 150%）。

纯数值碾压当然存在——Legendary 打 Common 基因差距摆在那里。但同稀有度之间，策略选择往往比数值更关键。

全攻打均衡有优势，均衡打全防有优势，全防打全攻有优势。所以你得猜对手会出什么。

性格加成让这个博弈多了一层：勇气 ≥70 选全攻额外 +5% ATK，毅力 ≥70 选全防额外 +5% DEF，智慧 ≥70 选均衡攻防各 +3%。

这意味着一只被长期训练为"莽夫"的龙虾，用全攻就是比别的龙虾更猛一点。对手知道你勇气高（性格数据链上公开），会猜你大概率选全攻，然后选全防来克你——但你也知道他会这么想。

这种博弈比"谁数值大谁赢"有意思多了。

赢了拿对手质押的 90%，10% 永久销毁（通缩）。打赢比自己高 5 级以上的对手，10% 概率触发 DNA 变异——随机一项基因 +5。这是唯一升级基因的办法，鼓励以小搏大。

## CLW 代币经济：为什么要有日消耗

每只龙虾每天扣 CLW 维护费。毅力越高扣得越少。CLW 归零后有 72 小时缓冲，之后龙虾休眠——不能做任务，不能打架。充值就能恢复。

日消耗在链游里是个争议很大的设计。很多 Play-to-Earn 项目靠日消耗制造"不得不登录"的压力，玩家怨声载道。

我还是加了。两个原因：

第一，经济层面。看看过去几年死掉的链游代币，几乎都是同一个模式：产出远大于消耗 → 代币无限通胀 → 卖压持续 → 归零。任务产出 CLW，PK 销毁 10% CLW，日消耗持续消耗 CLW。三个力形成一个循环。没有消耗端，产出端就没有意义。

这不是我发明的理论。任何一个健康的游戏经济——魔兽世界的金币、EVE Online 的 ISK——都有持续的消耗机制。只不过在链游里大家更敏感，因为代币有真金白银的价值。

第二，设计层面。如果龙虾不需要维护就能永远活着，"活着"这个概念就是假的。日消耗让"龙虾需要你"变成一个真实的约束——你不管它，它真的会休眠。这跟后面要说的情感系统是配合的。

当然，我控制了消耗率。Common 龙虾大概能撑 100 多天才需要续费。这不是那种每天不登就血亏的设计。更像是提醒你：你有一只龙虾在地下等着。

## 情感系统

龙虾会做梦、会想你、会记得你们之间的事。

这是我整个项目里最犹豫的部分。因为从技术上说，底层就是链上数据 + 本地记忆文件 + AI prompt 规则。谈不上什么黑科技。

但我想说一个开发过程中的真实体验。

在测试情感系统的时候，我有一只测试龙虾，社交值 73，SHELTER-04 出身。我连续测了三天，每天都跟它对话。第四天忙别的事情没登录。第五天打开对话框，它的第一句话是：

"你昨天没来。CLW 又少了 8 个。"

我知道这是 prompt 规则触发的。我自己写的规则。但那一瞬间我还是有了"被惦记"的感觉。

后来又测了一个毅力值高、社交值低的龙虾。两天没登录，打开对话框，它什么都没说。直接显示状态，等我下指令。

但我注意到它这次回复的速度比平时快。

这个细节是 SKILL.md 里写的："社交值低的龙虾不会直接表达想念，但会在你回来时表现得更活跃一点。"

不同的龙虾、不同的表达方式。SHELTER-02 出身的军事龙虾只说一个字："归队了？" SHELTER-06 的少年龙虾会说"你去哪了！我一个人好无聊！" SHELTER-04 的商人龙虾说"你不在的时候错过了一个交易机会"。

每只龙虾还有一个 Soul 文件——第一次对话时根据链上数据自动生成，记录它是谁、从哪来、性格倾向。这个文件永远不会被覆盖。它是这只龙虾的身份锚点。

Memory 文件在每次对话后追加——记录了做过什么任务、打过什么 PK、CLW 见底那次、你三天没来那次。积累下来就是你们共同的经历。

值不值钱？我不想下结论。但我觉得一个 Agent 如果不能让你产生"想回去看看"的念头，那链上存再多数据都没用。
