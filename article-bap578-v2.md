BNB Chain 去年发布了 BAP-578，NFA 标准。Non-Fungible Agent——让 NFT 从图片变成链上智能体的技术规范。

标准出来一年了，分析文章写了不少，但完整做出来的产品，据我所知还没有。

这篇文章讲三件事：NFA 到底要解决什么问题，我怎么实现的，以及你可以怎么玩。

## 先聊聊 NFA 是什么

大多数人理解的 NFT：一张图片的链上凭证。IPFS 存着一个 JSON，JSON 里指向一张 PNG。买了之后它就躺在那儿，等你卖掉。

BAP-578 想做的事情不一样。它给 NFT 加了四个能力：

**身份** — 链上唯一 ID，绑定所有者，可转移。这个传统 NFT 已经有了。

**钱包** — 每个 Agent 有自己的资产账户。代币不在你钱包里混着，在 Agent 自己的余额里。

**执行** — Agent 可以通过授权的合约执行操作。做任务、打架、交易，都是 Agent 自己在链上完成的。

**学习** — Agent 能进化，进化历史记在链上。

把这四个能力拼在一起，就不是一张图片了。

## 三种资产的区别

写到这里可能有人问：市面上已经有一些自称 NFA 的项目了，你的有什么不同？

说一下我观察到的三种实现层次：

**传统 NFT** 就不用说了。一张图。

**现有的 NFA 尝试** 大多做到了链上身份这一步——给 NFT 加了状态字段和逻辑地址。但 AI 部分通常是接了一个云端 API。你的 Agent 跑在项目方的服务器上，项目方关了服务器，Agent 就没了。说难听点，这是租了一个 Agent，不是拥有一个。

**我做的这个**，AI 跑在你自己的设备上。你装一个叫 OpenClaw 的本地 AI 框架，龙虾作为一个 Skill 插件运行。你连什么大模型自己选——Claude、GPT、Gemini、甚至本地跑的开源模型。我的服务器不参与推理过程，不知道你的龙虾说了什么。

这一点我觉得很重要。做 AI Agent 的项目，Agent 的命脉不应该在项目方手里。

| | 传统 NFT | 现有 NFA | CLAW NFA |
|--|---------|---------|----------|
| 链上数据 | 图片 URI | 身份 + 状态 | 身份 + 人格 + 基因 + 经济账户 + 学习树 |
| AI 在哪跑 | 没有 AI | 项目方服务器 | 你自己的设备 |
| 会进化吗 | 不会 | 有限 | 5 维人格 + DNA 变异 |
| 项目方跑路 | 图还在 | Agent 死了 | Agent 还活着 |

## 合约怎么写的

10 个合约，简单过一下各自干什么：

| 合约 | 一句话 |
|------|--------|
| ClawNFA | ERC-721 底层，存身份、元数据、学习树 |
| ClawRouter | 中心路由，管 CLW 余额，给 Skill 合约授权 |
| PersonalityEngine | 处理性格演化，带月度上限 |
| TaskSkill | 任务结算，算奖励，触发性格漂移 |
| PKSkill | PvP，commit-reveal 防作弊，性格加成 |
| MarketSkill | 交易市场，固定价/拍卖/互换三种模式 |
| GenesisVault | 创世铸造，commit-reveal + 自动分配图片 |
| DepositRouter | 买 CLW 充值，带滑点保护 |
| WorldState | 全局参数，24 小时时间锁防止随意修改 |
| ClawOracle | AI 预言机接口 |

全部用 UUPS 可升级代理。加新功能不需要迁移数据，升级合约就行。每个合约留了 40 个 storage slot 给未来扩展。

安全方面做了这些：铸造和 PK 用 commit-reveal 防前跑。市场退款用 pull-over-push 模式。提现有 6 小时冷却期。世界参数修改有 24 小时时间锁。operator 任务奖励有硬上限。做了一次内部审计，修了 5 个高危和中危问题。

235 个测试用例跑完大概 2 分钟，0 失败。

## 性格系统是怎么工作的

每只龙虾有 5 个性格维度：勇气、智慧、社交、创造、毅力。初始值在铸造时随机生成。

之后怎么变，看你怎么玩。

做冒险类任务 → 勇气 +1。做解谜类任务 → 智慧 +1。每月每个维度上限 ±5，做 5 次同类任务就到顶了。

关键在于——性格直接决定你赚多少。

任务有匹配度，算法很简单：那个维度的性格值 × 200 = matchScore。勇气 80 做勇气任务，matchScore 就是 16000，也就是 1.6 倍奖励。勇气 10 做同样的任务，2000，0.2 倍。

同一个任务，8 倍差距。这意味着专精培养的龙虾和随便玩的龙虾，收益完全不在一个量级。

所以"养龙虾"这件事就有了真实的决策成本——你选了冒险任务，勇气涨了，但其他维度的任务你就赚得少了。要不要全点一个方向？还是平衡发展？这个选择权在你。

## PK 怎么打

三种策略：全攻、均衡、全防。类似石头剪刀布的克制关系。

具体的战斗公式涉及 4 项基因（力量、防御、速度、体力），策略选择决定攻防倍率。速度高的先手，额外 10% 伤害。

有意思的地方在性格加成：

勇气 ≥70 的龙虾选全攻，攻击力额外 +5%。毅力 ≥70 选全防，防御力 +5%。智慧 ≥70 选均衡，攻防各 +3%。

用擂台模式的话，创建时选好策略，对手加入时也选好，链上自动结算。两笔交易搞定一场 PK。

赢了拿对方质押的 90%，10% 永久销毁。输了拿一半 XP。打赢比自己高 5 级以上的对手，10% 概率触发 DNA 变异——随机一项基因 +5。这是唯一升级基因的方式。

## 为什么要有每日消耗

龙虾每天扣 CLW 维护费。毅力高的扣得少。

余额归零 72 小时后休眠——不能做任务，不能打架。充值就能恢复。

我知道"日消耗"在链游里是个争议性设计。很多玩家讨厌它。

但我还是加了，原因是：没有消耗就没有经济循环。如果 CLW 只涨不花，代币没有真实需求，最后一定崩。日消耗 + PK 销毁，让 CLW 有了持续的消耗端，跟任务产出形成平衡。

另一个原因更主观一点——我希望玩家每天记得来看看龙虾。如果没有任何后果，你一个月不登录也没影响，那 Agent 的"活着"就是假的。消耗机制让"保持龙虾活着"变成一件需要在意的事。

## 关于情感

这部分我犹豫了很久要不要写进来，因为听起来很容易被当成噱头。

龙虾在你不在的时候会"做梦"。你两天没来它会表达想你。它记得你们之间的重要事件——第一次 PK 胜利、CLW 快见底那次、你选了一个完全不匹配的任务那次。

底层实现很简单：链上数据 + 本地记忆文件 + AI prompt 规则。没有什么黑科技。

但效果比我预想的好。测试的时候，我自己都会好奇"它今天说什么了"。明知道是 prompt 生成的，还是忍不住想看。

不同性格的龙虾表达方式完全不同。社交值高的直接说"你去哪了"。社交值低的什么都不说，只是你回来的时候响应快了一点。SHELTER-02 出身的军事龙虾说"归队了？"。SHELTER-04 的商人龙虾说"你不在的时候 CLW 又跌了"。

这些东西值不值钱我不确定。但我觉得一个 Agent 如果不能让你产生"想回去看看它"的念头，那 Agent 这个词就是空的。

## 没做到的事情

说一些这个版本的限制，省得你自己去发现：

**没有专业审计。** 做了内部审计，修了该修的，但没请 CertiK 之类的第三方。主要是成本。合约代码是开源的，欢迎 review。

**888 只很少。** 这是有意的。第一个版本想控制规模，验证机制跑通了再扩。

**AI 质量取决于你用什么模型。** 我测试用的 Claude，效果不错。但如果你接一个很弱的本地模型，龙虾说话可能不太聪明。这是本地运行的代价——自由度高，但质量不保证。

**代币经济没经过实战。** 日消耗率、任务奖励、PK 销毁比例这些参数，是我根据模拟数据设的。真实玩家进来之后大概率需要调。WorldState 合约支持 24 小时时间锁调参，预留了这个能力。
